# KANTAN AI Linux deb 包打包指南

## 📋 脚本说明

本项目包含多个自动化脚本，用于修复和打包 KANTAN AI 的 Linux deb 包。

## 🚀 快速开始

### 方法一：使用完整自动化脚本（推荐）

```bash
# 进入脚本目录
cd /home/kantan

# 赋予执行权限
chmod +x kantan-auto-build.sh

# 运行脚本
./kantan-auto-build.sh
```

### 方法二：分步执行

#### 1. 仅修复配置文件
```bash
chmod +x fix-config-only.sh
./fix-config-only.sh
```

#### 2. 一键构建（简化版）
```bash
chmod +x one-click-build.sh
./one-click-build.sh
```

## 📝 脚本功能对比

| 脚本名称 | 功能 | 适用场景 |
|---------|------|----------|
| kantan-auto-build.sh | 完整自动化修复和打包 | 首次打包或遇到问题时使用 |
| auto-build.sh | 完整自动化修复和打包 | 同上 |
| complete-build.sh | 完整自动化修复和打包 | 同上 |
| fix-config-only.sh | 仅修复配置文件 | 只想修复配置文件 |
| one-click-build.sh | 一键构建（简化版） | 快速构建，输出简洁 |

## 🔧 脚本执行步骤

### kantan-auto-build.sh 执行流程：

1. **修复配置文件**
   - 自动检测并修复 `electron-builder.json`
   - 将 `Name`, `Comment`, `Terminal` 等属性移动到 `entry` 对象

2. **安装编译工具**
   - 检查并安装 `build-essential`
   - 安装 Python3 和相关工具
   - 安装 `libtool`, `nasm`, `autoconf`, `automake`, `pkg-config`

3. **清理构建文件**
   - 删除旧的 `dist`, `dist-electron`, `release` 目录
   - 删除 `node_modules` 和 `package-lock.json`

4. **安装依赖**
   - 运行 `npm install` 重新安装所有依赖

5. **执行打包**
   - 运行 `npm run dist`linux` 打包 deb 包
   - 显示打包结果和 deb 包路径

## 📦 安装 deb 包

打包成功后，使用以下命令安装：

```bash
# 进入项目目录
cd /home/kantan/KANTAN-AI

# 安装 deb 包
sudo dpkg -i release/kantan-ai_2026.4.6_amd64.deb

# 如果遇到依赖问题，运行：
sudo apt-get install -f
```

## 🔍 常见问题

### 1. 权限问题
**错误信息**：`Permission denied`

**解决方案**：
```bash
chmod +x kantan-auto-build.sh
```

### 2. Node.js 版本过低
**错误信息**：`Node.js version too old`

**解决方案**：脚本会自动升级 Node.js，或手动运行：
```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -S bash -
sudo apt-get install -y nodejs
```

### 3. 编译工具缺失
**错误信息**：`node-gyp failed to rebuild`

**解决方案**：脚本会自动安装编译工具，或手动运行：
```bash
sudo apt-get install -y build-essential python3 python3-pip python3-distutils
```

### 4. 配置文件错误
**错误信息**：`Invalid configuration object`

**解决方案**：脚本会自动修复配置文件，或手动运行：
```bash
./fix-config-only.sh
```

### 5. 磁盘空间不足
**错误信息**：`No space left on device`

**解决方案**：清理磁盘空间
```bash
# 清理 apt 缓存
sudo apt-get clean

# 清理 npm 缓存
npm cache clean --force

# 清理旧的构建文件
rm -rf /home/kantan/KANTAN-AI/dist /home/kantan/KANTAN-AI/dist-electron /home/kantan/KANTAN-AI/release
```

## 📊 系统要求

- **操作系统**：Debian 11 或兼容的 Linux 发行版
- **Node.js**：>= 22.12.0（脚本会自动升级）
- **磁盘空间**：至少 2GB 可用空间
- **内存**：建议 4GB 以上
- **网络**：需要访问 npm 和 apt 源

## 🎯 最佳实践

1. **首次打包**：使用 `kantan-auto-build.sh` 完整脚本
2. **后续更新**：如果只是代码更新，可以直接运行 `npm run dist:linux`
3. **遇到问题**：重新运行 `kantan-auto-build.sh` 清理并重新构建
4. **查看日志**：构建日志会保存在 `build-output.log`

## 📞 获取帮助

如果遇到问题，请检查：

1. 查看构建日志：`cat build-output.log`
2. 检查 Node.js 版本：`node --version`
3. 检查 npm 版本：`npm --version`
4. 检查磁盘空间：`df -h`
